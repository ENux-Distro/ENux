## ENux Phase 0, Phase 1 and Phase 2

These 3 scripts, did this:

- ENux Phase 0: Optional, create .desktop shortcuts for easier access for the user
- ENux Phase 1: Converts your existing system to an ENux branded one with *Bedrock Linux installed*
- ENux Phase 2: *brl fetched* the stratas.

They were bundled in the ENux Beta 1.0 tar archive (excluding ENux Phase 0, that was an git exclusive)

## ENux *Finaliser* Phase 1 and Phase 2.

Yes, finaliser not finalizer. These were the ENux Phase 1 and ENux Phase 2 from Beta 1.0, which was bundled with the first ENux ISO release.
I thought that these scripts was incredibly powerfull and Bedrock Linux systems can be only installed like that. I was very wrong.

## ENux Finaliser

ENux Finaliser was one of the things of ENux that made me sad after removing this. The Finaliser got replaced with the ENux Welcomer. 

## ENux Installer CLI

I loved the ENux CLI installer.

**But every love story must end.**

Maintaining two installers at the same time became difficult, especially after the TUI installer looked really good on ENux Netinst.

I originally dropped the CLI installer from ENux Netinst 5.3.2, and later removed it entirely from ENux.

***I still miss it sometimes.***

## (Upcoming) ENux Welcomer (aka the ENux Firstboot logic)

Wow. I feel really sad because I'm dropping the ENux Welcomer and the firstboot logic. It served me from **Beta 1.0 to ENux 5.5.3*. 

*But some loved ones needs to be replaced.*

With my recent breaktrough of making ENux have 6 stratas and 15 package managers in the live environment, I realized that brl fetch operations can be taken
to the next level. I'm currently working on at ENux 5.6.3, which'll combine the Installer + Welcomer into a one powerhouse installer at the time of writing.
So a few days later, ENux Welcomer and the firstboot logic will be in the grave permanently.

*It had a good run*
